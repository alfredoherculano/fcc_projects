About

Git is a version control system originally developed by Linus Torvalds, creator of the Linux operating system. Version control is a way to record changes to a file or system of files over time so that you can recall specific versions later.

Installing Git

Installing on Linux

It's easier to install Git on Linux through the preferred packge manager of your distribution. For that, follow the specific information at the Git website.

Installing on macOS

On Mavericks (10.9) or above, symply run $ git --version from the terminal. If you don't have it installed already, it will prompt you to do it.

Installing on Windows

Follow the instructions at https://git-scm.com/download/win.

First Time Setup

You'll only need to do this one time for each machine.

First, you need to setup your username and email:

$ git config --global user.name "John Doe" $ git config --global user.email johndoe@example.com

To set a main as your default branch name you can run the following:

$ git config --global init.defaultBranch main

Finally, to check your configuration settings you can run:

$ git config --list
Getting a Git Repository

Initializing a Repository in an Existing Directory

In your terminal, navigate to the directory you want to turn into a Git repository and type:

$ git init

This creates a folder named .git that contains all the necessary repository files.
